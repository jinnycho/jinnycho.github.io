💣 Radiation Map of the World v1.0
Jul 21, 2019
5 minute read

Motivation

For the past decades, there have been a number of large-scale nuclear accidents around the world that devastated lives of millions of people. Especially as a Korean who was born after 1990, Fukushima Daiichi accident is by far one of the most suspicious and concerning nuclear accidents. This just happened in 2011 in Fukushima Prefecture, 220km away from Tokyo. (It is about the distance between DC and Philadelphia!) I heard that this accident was as severe (or worse) as Chernobyl and the number of people who are diagnosed with leukemia or cancer soared. However, strangely no one really knows what really happened and about 30 million tourists still go to Japan. There will be even Olympics in 2020. I was curious to see the world map and personally check which parts should I particularly be careful.

2. Data Processing

The size of the dataset is 16GB. This is way above the limit that Mapbox allows, which is 1GB. So I came up with my personal solution to visualize the data on the map.

  • 🍄 Spark - It is a framework we can use for programming distributed processing of large dataset very fast. It is known to be 100x faster than Hadoop MR in memory and DAG engine optimizes the workflow. Considering the size of the dataset & the possibility of growing, Spark seems like a good use of my time.
  • 🔮 Scala - There are a few languages that we can choose to use Spark. People tend to use Python for its familiarity in terms of programming syntax. However, after some research, I realized that Scala is a lot more efficient, fast and stable. So I decided to write in Scala.
  • 💎 Structured Data - There is a trend to use structured data more because of its efficiency, better interoprability (e.g. MLLib, Spark Streaming) and simplicity. So throughtout the course, I use SparkSession, DataFrames and SparkSQL.
  • 🌱 sbt: There are 3 dependencies in this project but it can be larger. So I tried to package up the dependencies using sbt. Using sbt, I can easily define the dependencies and their versions.
  • 💅 Data cleaning: There are a lot of properties that are not very necessary (e.g. location_name, unit, device_ID etc). I pruned all of those to make the dataset more compact. I also splitted the dataset into 5 parts so each is around 3GB instead of 15GB. This is to reduce the processing time. But this splitting is probably not necessary if you have more machines or time. Reducing to 3GB enabled me to receive the results in 15 mins and 3GB is still way above the limit that the corporate set.
  • ✨ MLLib: MLLib is an Apache Spark’s scalable machine learning library. I used K-Means Clustering because of multiple reasons. First, the dataset is too larget to render on the map. Just 1GB of data is already tens of millions of rows. Second, we have a lot of redundant radiation measurments in one same spot. In that case, we should have some logic to choose one. captured_time can be one of them (e.g. prefer more recent data) but it can be wrong (e.g. the device might be broken). Also it’s quite annoying to dedupe the data and sort by date. As a result, I think clustering is a good way to extract the truth in that region in a relatively quick way given such a large dataset.

3. Visualization

  • 🗺 Mapbox gl-js: I used Mapbox gl-js as a base map to render the radiation data. I liked that it has a light version where there’s no color, so I can see the radiation severity more quickly.
  • 🍹 D3: D3 has very cool visualization functions that work really smoothly with html, css and JavaScript. I used D3 to show points and values when hovered.
  • 🍓 Node.js, Express & JavaScript: Node.js + Express makes building web application framework so much simpler. Also JavaScript is a nice language for web pages that also works well with Mapbox gl-js, D3 and Node.js.

Thoughts

About the map

  • Unsurprisingly many parts of Japan is contaminated. Especially near Fukushima, the radation value (colored black) is above the warning level (which is above 100).
  • Korea also have some locations with radioactivity measurements (not as a severe & frequent as Japan but still). There was no known nuclear accidents. My hypothesis is I remember Korea imported a lot of sands & trash from Japan when they built apartment buildings and roads. I wonder if the sand they imported is actually from Fukushima area.
  • Europe also has a lot of radiation values measured very extensively across the continent. The severity of radiation is not that high compared to Japan, but it’s quite surprising that there are such a lot of measurements.
  • The map is not 100% accurate because
    • it used clustering to extract the essense of the data. So depending on the number of clusters, the shape of the map can potentially change.
    • It can be also quite biased because these measurements depend on the number of people who have the device that measures the radiation values. So even if there’s not much radiation in China, that doesn’t necessarily mean there’s no radiation but it could be just that there’re not enough people that have the device to measure radiation values.

Improvements

  • It’d be cool to see how radiation measurements change by year or month.
  • I can try automating the process of updating the dataset.
